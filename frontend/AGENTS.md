# AGENTS.md

## Общие правила

- Используй `React + TypeScript` и функциональные компоненты.
- Держи код в рамках FSD-структуры: `app`, `pages`, `features`, `entities`, `shared`.
- Не добавляй лишнюю сложность: backend, API-обёртки и дополнительные абстракции только если они действительно нужны задаче.
- Предпочитай небольшие переиспользуемые компоненты и чистые хуки вместо больших файлов с плотной логикой.

## UI и стили

- Основной способ стилизации: `Tailwind`.
- Глобальные токены, базовые стили и кастомные CSS-классы держи в `src/index.css`.
- Сохраняй текущий визуальный язык проекта: светлая палитра, мягкие карточки, зелёные акценты, округлые формы.
- Если меняешь общий layout, не дублируй его на страницах. Общие элементы должны жить в `app/layout`.

## Состояние и данные

- Глобальное клиентское состояние для MVP держи в `zustand`.
- Основной стор проекта находится в `src/features/game/model/useGameStore.ts`.
- Для демо и MVP используй `mockData`, а не новые удалённые API.
- Если появляются координаты, маршруты или другие навигационные данные, выноси их в отдельные конфиги/модели, а не хардкодь внутри UI-компонентов.

## Маршрутизация

- Маршрутизацию держи через `react-router-dom`.
- Роутер описывай в `src/app/router.tsx`.
- Страницы не должны содержать глобальный хидер/футер внутри себя, если они уже есть в `AppShell`.

## Карта и геолокация

- Логику карты держи в `features/navigation`.
- Геолокацию получай через `shared/lib/useCurrentGeolocation.ts`.
- Точки маршрута и destination ids храни в `features/navigation/model`.
- Маршруты по карте должны строиться через конфиг точек, а не через вшитые координаты в JSX.

## Качество изменений

- После изменений удаляй неиспользуемые компоненты, файлы и ассеты.
- Перед завершением задачи прогоняй как минимум `npm run lint` и `npm run build`, если изменения затрагивают приложение.
- Если обновляешь архитектуру проекта, синхронизируй `AGENTS.md` и `ARCHITECTURE.md`.
