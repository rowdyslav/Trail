# Architecture

## Обзор

Проект — это клиентское MVP-приложение на `React + TypeScript + Vite` с FSD-подобной структурой. Основной сценарий: пользователь открывает главную страницу, переходит в маршрут, использует карту и сканирование, получает награды и отслеживает прогресс в профиле.

## Слои

### `src/app`

- `App.tsx` — корневой React-компонент приложения.
- `router.tsx` — описание маршрутов через `react-router-dom`.
- `layout/AppShell.tsx` — общий shell приложения: верхняя навигация, нижняя навигация, общие overlay-компоненты.

### `src/pages`

- `home/HomePage.tsx` — главная страница с промо-блоками и CTA.
- `route/RoutePage.tsx` — страница активного маршрута, карта и список точек.
- `profile/ProfilePage.tsx` — страница прогресса пользователя и наград.

### `src/features`

- `game/model/useGameStore.ts` — главный zustand-store приложения.
  Отвечает за пользователя, маршрут, сканирование и награды.
- `navigation/model/routePoints.ts` — конфигурация точек маршрута и destination ids.
- `navigation/ui/RouteMap.tsx` — карта на `react-leaflet`, загрузка маршрута, маркеры, легенда.
- `scan/ui/ScanOverlay.tsx` — overlay сканирования.
- `rewards/ui/RewardModal.tsx` — модальное окно награды.

### `src/entities`

- `quest/model/mockData.ts` — моковые данные маршрута, пользователя, профиля и наград.

### `src/shared`

- `lib/cn.ts` — helper для объединения CSS-классов.
- `lib/useCurrentGeolocation.ts` — кастомный хук для чтения текущей геолокации устройства.
- `types/game.ts` — общие доменные типы.
- `ui/Button.tsx` — базовая кнопка, используемая в общих сценариях.

## Маршрутизация

Роуты строятся вокруг `AppShell`:

- `/` — главная страница
- `/route` — страница маршрута
- `/profile` — профиль

`AppShell` оборачивает все страницы и содержит общий header, bottom navigation, `ScanOverlay` и `RewardModal`.

## Поток данных

### Пользователь и прогресс

- Данные пользователя и маршрута инициализируются из `mockData`.
- Состояние хранится в `useGameStore`.
- При сканировании store обновляет:
  - статус checkpoint
  - общий прогресс маршрута
  - XP пользователя
  - уровень и стадию аватара
  - активную награду

### Геолокация и карта

- `useCurrentGeolocation` получает текущие координаты пользователя.
- `RouteMap` использует эти координаты как стартовую точку.
- Финиш выбирается по `destinationId` из `routePoints`.
- Маршрут загружается через OSRM и рисуется на карте как `Polyline`.
- Если роутинг недоступен, карта показывает упрощённую прямую линию как fallback.

## Принципы проекта

- Общий layout живёт в `app/layout`, а не копируется по страницам.
- Бизнес-сценарии хранятся в `features`.
- UI не должен содержать жёстко вшитые координаты или конфиги маршрута.
- Для MVP предпочтительны простые client-side решения и mock data.
